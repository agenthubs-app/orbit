## Sprint Definitions

- **Sprint 81 - Relationship Database Schema Optimization:** Evaluate the advisory relationship-design document against the current `repos/orbits` local-remote database, then evolve the schema with minimal redundancy and high query value. Preserve existing product DTOs, add event-specific intent and relationship scoring where they need top-level queryability, add semantic AI/recommendation/interaction collections where JSON is the right boundary, bump schema version, update docs, and prove the shape with focused tests.

- **Sprint 82 - MiniMax AI Relationship Mockdata Generation:** Use a project-root AI agent backed by the MiniMax Anthropic-compatible Claude SDK to generate v1 demo-scale relationship seed data under `repos/mockdata`. The data must follow the design-document scale and ratios, include the ten required demo scenarios, produce dictionaries/seed/tests/scenarios/generated/exports files, and include deterministic validation evidence for counts, references, Japanese/Chinese language mix, golden matches, negative cases, and dirty data coverage.
